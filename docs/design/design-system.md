## 1. デザイン原則（コンセプト → 視覚の翻訳）

concept の提供価値「数学の学びなおし × テックツリーの到達感 × 挫折者に優しい × AI 検証の信頼感」から導出:

1. **導いてくれる、威圧しない** — つまずいた人が安心して戻れる。深く落ち着いた背景で、要素を詰め込まず余白で呼吸させる。（ターゲット=挫折経験者）
2. **到達が見える** — テックツリーの「解放されていく」高揚感を上品に表現。状態（習得/解放/未解放）が一目で分かり、進捗が報酬になる。（差別化=アンロック可視化）
3. **知的で信頼できる** — 藍を基調に数学の落ち着きと信頼を表す。AI 生成でも正確性を担保するプロダクトの真面目さを視覚に乗せる。
4. **数式と調和する** — KaTeX の数式描画が主役の場面が多い。タイポ・コントラストは数式の可読性を最優先。
5. **煽らない（charter §2.2）** — ランキング・ガチャ・FOMO の視覚言語を持たない。アンロックは確定的な解放の喜びのみ。

### 2.1 ノード状態カラー（テックツリーの核）
| 状態 | 表現 |
|---|---|
| **習得済** | `--primary` 充塗 + `--accent-soft` の到達リング |
| **解放済（次に学べる）** | `--primary` 輪郭 + 淡い内側グロー |

| **ロマンノード（遠いゴール）** | `--accent` 輪郭の点線 + 「遠景」感（低不透明 + 小さめ） |

## 3. タイポグラフィ

| 役割 | 指定 | 備考 |
|---|---|---|
| 本文（和） | システムゴシック（Hiragino/Noto Sans JP 系） | 可読性最優先 |
| 本文（欧/数字） | 中立サンセリフ（Inter 等） | KaTeX と調和 |
| 数式 | KaTeX 既定（Computer Modern） | 描画のみ、`trust:false, strict:true` |
| 見出し | 同上 sans、`font-weight:600` | 装飾過多にしない |

スケール: `12 / 14 / 16(base) / 20 / 24 / 32`。行間 本文 1.7 / 見出し 1.3。

## 4. 形・影・余白

- **角丸**: `--radius-sm 6px` / `--radius 10px`（カード）/ `--radius-lg 16px`（モーダル）/ ノードは `--radius 10px`〜pill
- **影**: `--shadow-1`（カード: 0 1px 3px rgba(0,0,0,.4)）/ `--shadow-2`（浮き: 0 8px 24px rgba(0,0,0,.5)）。多用しない。
- **余白スケール**: `4 / 8 / 12 / 16 / 24 / 32 / 48`。要素は詰め込まず呼吸させる（原則#1）。

## 5. コンポーネント（基本形）

- **ボタン**: primary（藍充塗）/ accent（琥珀=主要 CTA・支援・次へ）/ ghost（境界のみ）。最小タップ 44px。
- **カード**: `--surface` + `--border` + `--radius` + `--shadow-1`。
- **チップ**: 状態タグ（習得/解放/未解放、豆知識カテゴリ）。状態色トークンを使う。
- **入力/数式エディタ**: MathLive を `--surface-raised` の枠に収め、フォーカスで `--primary` 境界。
- **ヘッダー/ナビ（モバイル横一列の標準形, O61）**: アプリ名（左、`white-space:nowrap` + `text-overflow:ellipsis`）+ コンパクトメニュー（右、`flex-shrink:0`）。狭幅で縦折れさせない。収まらなければアイコン化/ボトムナビへ退避。
- **入口の「これは何？」導線（O41）**: トップに簡潔なリード文（「数学を初歩からテックツリーで学びなおす」）。収まらなければ丸付き「?」インフォボタン + 軽量モーダル。
- **進捗体験（O45）**: AI 生成は事前生成のため実行時の本質遅延は小さいが、CAS/AI 同値判定や生成待ち画面では「実段階に紐づく文言 + 軽い動き」を出す（嘘進捗・わざと遅延は NG）。
- **支援（tip-jar）導線（O43, 課金 PJ）**: 「100 円で作者を支援」と**金額 + 対価を CTA 前・ファーストビューで明示**。煽らない常設配置。

## 6. ボイス & コピー（O38 準拠）

- ユーザー向け文字列に**技術用語を入れない**（NG: 「ノードをアンロック」→ OK: 「この単元が解放されました」/ NG: 「バリデーションエラー」→ OK: 「式を確認してください」）。
- トーン: 穏やか・端正・導いてくれる。挫折者を急かさない。正解時も大げさに煽らない。
- エラー: 責めない。「惜しい、ここを見直してみましょう」。
- 詳細なトーン校正（O42）は画面実装後に `/flow:wording` で対話校正 + 二言語整合（将来）。

## 7. アイコン & イラスト & ブランドマーク戦略

- **アイコン**: OSS セット（lucide 等）。**絵文字は使わない**（環境依存で崩れる）。
- **イラスト**: 自作 SVG line-art（空状態 = まだ何も学んでいないツリー、ヒーロー = 育つテックツリー）。テーマ色追従。
- **ブランドマーク（public-facing/PWA 必須、deferred to 適用フェーズ）**: 単一 SVG ソース（藍地に琥珀のノードが連なる小さなツリー/ルート記号 √ のモチーフ案）→ favicon.svg/.ico + apple-touch-icon 180 + icon-192/512 + maskable-512 + manifest.json icons + HTML head wiring。`scripts/gen-favicon` で派生生成。**絵文字 favicon は NG**。

## 8. レビュー基準（視覚 + コピー、適用フェーズで適用）

- 各画面を 360px モバイル幅でスクショ → SoT 適合（階層/余白/コントラスト/主役の明確さ/トーン）。
- 水平行の折り返し崩れ（ヘッダー縦折れ）チェック（O61）。
- token-conformance: 生値直書きなし、共通コンポーネント使用（原則#3）。
- コピー O38 deny-list 走査。入口「これは何？」理解（O41）。tip-jar 価格透明性（O43）。ルート到達性（O55, orphaned page なし）。
