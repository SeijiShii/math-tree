# 根本原因分析: 問題が表示されない
## 1. 5 Whys
| # | 問い | 答え |
|---|---|---|
| Why1 | なぜ問題が出ないか | WorkbookView が問題を fetch も表示もしない（slug + 空入力のみ） |
| Why2 | なぜ fetch しないか | 問題取得 API（SPEC §6.1）が存在しない → 呼びようがない |
| Why3 | なぜ API が無いか | tdd 実装で grade-step（採点）は作ったが、問題取得 endpoint + 表示配線が実装されなかった |
| Why4 | なぜ漏れたか | SPEC §6.1 に endpoint は記載があったが、「採点が動く」ことを完了基準にし「問題を見て解く」導線の到達性を検証しなかった |
| Why5（根本原因） | なぜ検証されなかったか | **学習の中核導線（問題表示→入力→採点）の到達性テスト/視覚確認が計画になく、問題表示の欠落が unit/build green をすり抜けた**（O57 系・核心 read-path 未配線） |
## 2. 直接原因
| ファイル | 問題 |
|---|---|
| api/（problem 取得 endpoint 不在） | SPEC §6.1 の問題取得が未実装 |
| src/features/learning-workbook/WorkbookView.tsx | 問題 fetch / 表示が皆無（slug + 空入力のみ） |
## 3. 根本原因
学習中核 read-path（問題表示）の取得 API + 表示が SPEC にあるのに未実装。採点 green を完了基準にし、問題表示の到達性を検証しなかった。
## 4. 寄与要因
| 種別 | 内容 |
|---|---|
| テスト不足 | 問題表示の到達性テストが無い |
| 実装漏れ | SPEC §6.1 endpoint 未作成 |
