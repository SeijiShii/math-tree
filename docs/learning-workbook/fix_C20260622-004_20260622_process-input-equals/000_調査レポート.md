# バグ/改修調査: 入力欄が分かりづらい + 途中式（イコール）が採点されない
> issue: C20260622-004 / process-input-equals / severity: medium / 再現可
## 1. 症状
(1) 解答入力欄が白box で「入力する場所」と分かりづらい。(2) 「-3+5=2」のように計算の途中経過をイコールで繋いで入力すると「自動採点の準備中…展開・整理した形で」という意味不明なコメントが出て採点されない。
## 2. 期待挙動（要件）
計算の途中経過を書きながら学べる（concept UC#5 途中式入力）。**途中式はイコールで繋いで書くのが自然**で、それを採点できる必要がある（ユーザー要件）。入力欄は明確に「入力する場所」と分かる。
## 3. 再現
/learn/:slug で「-3+5=2」と入力 → 採点されず混乱メッセージ。入力欄が白box。
## 5. 影響範囲
全学習ユーザー / 中核の学習体験。データ影響なし。
## 10. 仮説
(a) 入力 math-field に design-system §5 のテーマ未適用（白box）。(b) areEquivalent が「=」を含む式を parse できず null → 「準備中」案内。イコールチェーンの採点ロジックが無い。
