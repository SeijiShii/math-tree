# クレーム調査レポート
**claim id**: C20260622-006 / 実施日: 2026-06-22 / 対象機能: learning-workbook / 緊急度推定: medium（MVP は動くが学習の質・量が不足）
## 1. クレーム原文
```
1つの単元で1つしか問題がないのは少なすぎる
しかも毎回同じ問題
簡単な問題であれば10用意する。準備してあるたくさんの問題からランダムに選ぶ
次の単元に進むための合格率を設定する（60点など）
問題を解きながら現在の点数が分かるように
```
## 2. 分解結果
### 2.1 期待 (Expected)
- 1 単元に複数問題（簡単な単元なら 10 問程度）を用意。
- 準備した問題プールから**ランダムに出題**（毎回同じにしない）。
- 次の単元へ進むための**合格率（例 60 点）**を設定し、それを満たしたら習得＝アンロック。
- **解きながら現在の点数**が分かる（ライブスコア表示）。
### 2.2 現実 (Actual)
- 1 単元 = 先頭 problem 1 問のみ（seed も 1 問）。毎回同じ。
- 1 問正解で即習得（合格率の概念なし）。
- 点数表示なし。ランダム出題なし。
### 2.3 発生条件
学習画面（/learn/:slug）。常時。
### 2.4 影響範囲
全学習ユーザー / 学習の質・反復量。MVP の体験ループは動くが浅い。
### 2.6 報告者文脈
複数問を解いて理解を確かめ、合格基準を満たして次へ進みたい。
