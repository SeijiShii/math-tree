### 3.2 Medium / Low 詳細

#### [AUDIT-perspective-001] O56 favicon が完全に不在（Medium）
- **カテゴリ**: #4 観点反映
- **検出根拠**: `index.html` に `<link rel="icon">` が無く、`public/manifest.json` の `"icons": []` が空、`public/` に favicon アセット（svg/png）が存在しない。ブラウザタブはデフォルトアイコン表示。grep+実在チェック（step 3.5）併走で確定。
- **推奨修正アクション**: design-system「学びの樹」モチーフに沿った自作 SVG favicon を `public/favicon.svg` に作成 → `index.html` に `<link rel="icon" href="/favicon.svg" />`（+ apple-touch-icon）→ `manifest.json` `icons[]` に登録。Class A・no-key。`/flow:revise _shared/ui`（or 直接配線）。

#### [AUDIT-perspective-002] O64 Stripe webhook が 501 stub（Low / deferred）
- **カテゴリ**: #4 観点反映（detect-gated: tip-jar=金銭授受 + Stripe webhook 実在 → 評価必須）
- **検出根拠**: `api/support/webhook.ts` が `res.status(501)` を無条件返却（署名検証未 inject）。O64 の 3 状態分離（invalid_signature=4xx / irrelevant=200ack / ok）は未実装。ただし `src/features/support/webhook.ts` に `handleWebhook` ロジックは存在し、route stub が bypass している状態。
- **判断**: tip-jar 自体が `VITE_ENABLE_TIPJAR` で非表示（ゲスト専用 MVP のスコープ外、SCENARIO §5 と整合）。Stripe 実キー未配線。**release 工程で tip-jar 有効化する際に handleWebhook + 実 Stripe 署名検証を配線する deferred 項目**。現 MVP のブロッカーではない。
- **推奨修正アクション**: tip-jar 有効化時（`/flow:release` で Stripe live + 特商法）に webhook route を O64 準拠（3 状態分離）で配線。それまでは 501 で honest stub のまま据え置き可。
